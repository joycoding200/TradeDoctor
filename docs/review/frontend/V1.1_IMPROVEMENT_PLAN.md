## 二、按页面归纳的发现（基于截图实证）

### 2.1 Landing（未登录）`01_landing.png` / `10_mobile_landing.png`

✅ **优点**
- 两列布局（文字 + 内嵌 AuthTabs）合理
- 渐变文字 "TradeDoctor" 醒目
- 4 个信任项图标对齐
- 移动端切换为单列，AuthTabs 嵌入正确

⚠️ **问题**
1. **CTA "开始分析" 在未登录态下也可点击**（移动端可见），点击后被 ProtectedRoute 拦到 `/login`——属于"虚假 CTA"，**应直接锚到登录/注册 tab**
2. **移动端 Trust 卡片距底部留白过大**（"你的数据，你做主" 之后 800px+ 空白），建议增加一条"已为 N+ 用户生成报告"之类的小社会证明
3. 移动端展开汉堡菜单后**看不到关闭按钮的视觉反馈**（按钮颜色变化太弱）

### 2.2 Register / Login `02_register.png` / `02b_register_after.png` / `03_login.png`

✅ **优点**
- 密码强度条（弱/一般/中等/强/很强）视觉好
- Toast 错误提示有引导链接（"已有账号？登录"）
- 8 位 + 字母 + 数字 校验清晰

⚠️ **问题**
4. **占位符不一致**：注册页是"请输入邮箱"+"密码（至少8位，含字母+数字）"，登录页是"邮箱或手机号"+"密码"——视觉与文案应统一（建议注册页也用"邮箱或手机号"）
5. **前端注册 `email.includes("@")` 校验过于宽松**——`a@.com` 也能通过，HTML5 `type="email"` 会拦截部分畸形值，但脚本 `audit_xxx@audit.local` 居然触发了"邮箱格式不正确"提示（见 `02b_register_after.png`），说明 **`type="email"` 校验和 React state 不同步**——受控组件应在错误提示上更明确，并改用更可靠的正则
6. **注册页**没有"昵称"输入框（实际 `useAuth.login(token)` 后没有 nickname），登录后右上角显示"未设置昵称"——**应在注册时强制要求昵称**
7. 注册成功后**没有"验证邮箱"提示**——内测阶段可接受，正式上线前需补
8. **手机号模式未公开入口**：未在登录页提供"用手机号"切换 tab（注册页有，登录页没有）
9. 登录错误 "账号或密码错误" 时提示应该**不要具体区分**（"账号不存在"和"密码错误"会泄露账号存在性）——当前 LoginForm.tsx 第 61-70 行根据错误文案显示不同引导链接，泄露了账号是否存在

### 2.3 Upload `04_upload.png` / `05_upload_*.png`

✅ **优点**
- 4 步进度条（上传/解析/导入/分析）非常清晰
- 拖拽 + 点击选择双支持
- 卡片式"使用提示"展示技术细节
- 上传成功后自动跳转到 `/analysis/:id`

⚠️ **问题**
10. **进度条第 4 步的"4"是数字，前 3 步是对勾**——视觉风格不统一（应统一为对勾或序号）
11. **`使用提示` 卡片长期展示**——上传过一次后没有折叠，应在有文件后隐藏
12. **没有"已上传的最近一次文件"提示**——用户多次上传后不清楚是否重复
13. **拖拽时整个 dropzone 高亮反馈**没有 + 边框动画，状态变化太轻
14. **没有拖拽格式错误反馈**——拖入 `application/pdf` 时 dropzone 不会变红

### 2.4 Analysis 面板 `06_analysis.png` / `06b_stats_tab.png` / `06_analysis_tab_*.png` / `07b_history_first.png` / `10b_mobile_analysis.png`

✅ **优点**
- 净值曲线（EquityCurve）漂亮，PC 端 100% 清晰
- 4 个核心 KPI 卡片（总盈亏/胜率/最大回撤/完整交易）信息密度好
- 归因分析 4 维度齐全
- WhatIf 止损模拟有 5% 止损 + "假设每次开仓设 X% 止损线" 假设说明
- "诊断结论"高亮显示
- "环境 × 行为" 交叉分析卡片

⚠️ **问题（**最严重的一组**）**
15. **顶部黄色警告 banner 抢占视觉重心**："检测到 13 笔卖出对应的买入发生在交割单起始日期之前，持仓成本未知，已标记为盈亏=0。建议导入更早期的交割单以获得完整分析"——这条对**只买了第一个月**的散户是常态，但提示词**默认假设用户应该补数据**，让用户自责。**应改为中性提示 + 主动解决方案**：
    - 把 banner 改为"折叠徽章"，默认收起
    - 提供"我有更早的交割单 → 点击上传"按钮（已经会跳到 `/upload?attach_to=:id`）
    - 改文案为"检测到 N 笔持仓起始于交割单外，盈亏标记为 0"（去掉"建议导入更早期"，改成"如需准确分析，请导入更早数据"）
16. **未登录访问 `/analysis/:id` 直接被 ProtectedRoute 跳走**——如果用户从邮件链接过来会丢上下文，应保存 redirect= 参数
17. **直接访问 `/analysis`（无 id）显示 404**——预期应该有"分析列表"或提示，但目前是 404 (`06_analysis_no_data.png`)，**路由缺失**：
    - 后端 `listAnalyses` 已经存在
    - 应当补 `/analysis` 路由 → 重定向到 `/history` 或显示"请先选择分析"
18. **直接访问 `/report`（无 id）也是 404**——同上
19. **股票维度表格在 PC 端可以滚动**（截图 06 显示），但**移动端最后一列（总盈亏/平均持仓天数）被裁剪**——需要加 `overflow-x: auto` 包装或重排为卡片视图（见 `10b_mobile_analysis.png`）
20. **PC 端核心 KPI 卡片在 1024-1280 视口下会挤**（4 个卡片总宽 1024px+padding）——应是 `xl:grid-cols-4 md:grid-cols-2` 而不是固定 4
21. **顶部右侧"添加交割单"+"生成 AI 报告"按钮在移动端错位换行**（见 `10b_mobile_analysis.png`），应改为图标按钮（带 tooltip）
22. **股票代码只有 6 位数字，没显示股票名称**（600550, 300189 等）——**强烈建议显示名称**（如"北方华创"），让用户直观识别
23. **"赚钱靠这些" 和 "亏钱因为" 标签下，标签条没有进度条或对比**——只能看到金额，**没有对比"贡献比例"的可视化**（虽然底部有"贡献了总收益的 29%"文字，但视觉权重太弱）
24. **WhatIf 标签 "5% 止损线 / 10% 止损线 / 15% 止损线" 应当是 slider 拖动**，目前是固定 5% 一档——可以让用户自定义止损
25. **"展开详细数据（盈亏比、单笔明细、持仓分析）" 折叠区没有预览**——用户不知道里面有什么，点击意愿低
26. **`完整交易 42 笔（22 笔赚钱、7 笔亏钱、样本量足够分析）`**——分母 22+7=29 ≠ 42，应该说"已完成建仓 22 笔..."，**当前文案逻辑不清晰**
27. **"胜率 76%"**——分母是 22 还是 42？应该明确"胜率 = 赚钱笔数 / 完整交易笔数"，并显示分子分母
28. **总盈亏 "+1.9万"**——单位混乱（有的用"万"，有的用"元"），建议统一保留两位小数 + 单位

### 2.5 AI Report `07_report.png`

✅ **优点**
- Markdown 渲染质量好
- 「诊断日期」「核心结论」「优势清单」「风险警示」「改善建议」结构清晰
- 数据支撑 + 行为溯源 + 执行建议 三段式

⚠️ **问题（**关键 UX 问题**）**
29. **"帮助改进 TradeDoctor" 模态框在用户刚打开报告时立即弹出**——这是 **F 型阅读杀手**：
    - 用户点"生成 AI 报告"是**为了看报告**
    - 报告刚加载好就被 consent 弹窗挡住（见 `07_report.png`，报告文字几乎全在弹窗后面）
    - **应改为"报告读完后再询问"**（例如用户滚到 80% 时弹，或点击"分享到案例库"按钮时弹）
30. **下载按钮在右上角"返回分析面板"链接旁**——返回链接应该比"下载"更显眼（"下载" 是次要 CTA，"返回" 是上下文切换），但目前视觉上"下载报告"按钮更突出
31. **报告没有"目录"或"折叠"**——内容超过 5 个章节（诊断日期/核心结论/优势清单/风险警示/改善建议），用户找信息困难
32. **没有"复制到剪贴板"按钮**——很多用户会想复制报告片段
33. **"风险警示"的"严重等级"（高危/注意）用 emoji 表示**——可访问性差（屏幕阅读器读 emoji 不友好）

### 2.6 History `07_history.png`

✅ **优点**
- 卡片化设计，每条记录含文件名 / 日期范围 / 创建日期 / 文件数
- 右侧 "AI 报告" / "分析面板" 双 CTA
- hover 状态（边框/阴影）有反馈

⚠️ **问题**
34. **没有删除单个分析的入口**——只能"清空全部数据"。**应该支持按 ID 删除单条**（带二次确认）
35. **没有搜索/筛选**——文件名多了之后翻不到目标
36. **没有按时间排序切换**——目前按创建时间倒序是硬编码的
37. **空状态只显示"暂无分析记录"和"上传交割单"按钮**——没有"导入示例数据"或"查看 AI 报告示例"链接

### 2.7 Admin `08_admin.png`

✅ **优点**
- 独立登录页，token 与普通用户隔离
- 简洁的"管理员账号/密码"表单

⚠️ **问题**
38. **Admin 入口对用户隐藏**（Layout 不显示）——但**生产环境管理员页存在**意味着**安全风险**：任何人访问 `/admin` 都能看到登录页，可以通过爆破字典尝试。**应当**：
    - 把 `/admin` 改为不可猜测的路径（如 `/admin-7c2b9e/login`），或
    - 增加 IP 白名单，或
    - 至少加 captcha 防爆破

### 2.8 404 `09_404.png` / `10_404.png`

✅ **优点**
- 简洁的放大镜图标 + 404 + 返回首页
- 移动端和 PC 端一致

⚠️ **问题**
39. **没有搜索/常用链接引导**——404 是用户迷路的信号，应该给出"热门目的地"（分析/历史/上传）
40. **图标 1.0 颜色和背景对比度低**（暗色模式下偏暗），可能达不到 WCAG AA

### 2.9 移动端 `10_mobile_landing.png` / `10b_mobile_analysis.png` / `11_mobile_landing.png`

⚠️ **问题**
41. **核心 KPI 卡片 2×2 布局在小屏幕下过大**（见 `10b_mobile_analysis.png`），文字行高 1.5 + padding 12px，单卡占屏 200px
42. **表格溢出**：股票维度盈亏在 ≤480px 下需横向滚动，但当前没有视觉提示
43. **Hero 文字字号过大**（`text-5xl` 64px），移动端 `text-4xl` 36px 仍然比最佳实践（24-30px）大
44. **汉堡菜单展开后没有"我的报告/上传"等明确 CTA**——只有"新分析 / 历史报告 / 清空数据 / 退出"，没有"我的昵称"和"设置"
45. **没有"返回顶部"按钮**——长页面滚到底回不去
46. **Landing 移动端的 AuthTabs 卡片高度**大（≥600px），首屏看不到 Trust 卡片
47. **核心 KPI 卡片的"完整交易 42 笔"数字**在移动端 + 数字较大时下沿贴底（`mb-0`），应该有更大留白

### 2.10 暗色模式 `12_dark_landing.png`（脚本中虽访问但截图策略不严，逻辑验证从代码看）

✅ 通过代码侧验证：Tailwind `dark:` 类、`text-text-primary` 等 token 化已实现
⚠️ **建议补充**
48. **没有显式切换器**——用户不能主动切到深色（`Layout.tsx` 没有切换按钮）
49. **应该尊重系统 `prefers-color-scheme`**
