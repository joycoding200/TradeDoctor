# PRD: 短期优先级冲刺 — 净值曲线 + 股票维度盈亏 + AI Prompt 扩充

**Status**: Approved  
**Author**: Alex (PM Agent)  
**Date**: 2026-06-23  
**Sprint**: Week 1-2  
**Source**: 产品评测报告_2026-06-23.md (投资研究员评测)

---

## 1. Problem Statement

投资研究员评测打分 8/10，核心引擎已达行业主流水平，但三个 P0 短板阻碍产品从"可用"到"好用"：

1. **没有净值曲线图** — 最基础的视觉工具，TradesViz/Edgewonk 都有，散户看不到资金曲线走势
2. **没有股票维度盈亏** — 散户最基本的问题"哪只股票赚了亏了"无法回答，只能逐行翻 Positions 表
3. **AI Prompt 信息量严重不足** — 只发送 5 个字段（trades/win_rate/pnl/holding_days/patterns/whatif），AI 不知道用户最大回撤是 30% 还是 3%，无法针对具体交易给建议

## 2. Goals & Success Metrics

| Goal | Metric | Target |
|------|--------|--------|
| 补齐视觉化短板 | 净值曲线图在统计概览页渲染 | 上线即可见 |
| 回答"哪只股票赚了亏了" | 股票维度盈亏汇总表 | 按盈亏排序可见 |
| AI 报告深度提升 | AI Prompt 携带字段从 5 → 15+ | 诊断报告含回撤/MAE/止盈效率分析 |

## 3. Non-Goals

- 不做时间线叙事（中期）
- 不做行动计划机制（中期）
- 不做跨维度交叉分析（中期）
- 不做日历热力图/滚动窗口/基准对比（长期）

## 4. Implementation Tasks

### Task 1: 净值曲线图 (Equity Curve)

**Backend** — `backend/app/api/analysis.py` + `backend/app/schemas/analysis.py`

在 `get_stats()` 中，已有 max drawdown 计算遍历了 `sorted_positions` 的累计 PnL。复用该循环，收集每个 position 的 `{date, cum_pnl}` 数据点。

- Schema: 在 `StatsResponse` 新增 `equity_curve: list[EquityPoint]`
- `EquityPoint`: `{date: str, cum_pnl: float, cum_pnl_pct: float}`
- 起点为 `{首笔 exit_date, 0.0, 0.0}`，后续累加

**Frontend** — 新建 `frontend/src/components/EquityCurve.tsx`

- 使用 Recharts `AreaChart`（已在技术栈中）
- X 轴: 日期，Y 轴: 累计盈亏金额
- 颜色: 盈利区域绿色半透明，亏损区域红色半透明
- 放在 StatsTab 核心结果卡片上方

### Task 2: 股票维度盈亏表 (Symbol Summary)

**Backend** — `backend/app/api/analysis.py` + `backend/app/schemas/analysis.py`

在 `get_stats()` 中，从 `valid_positions` 按 `symbol` 分组，计算每只股票：
- `symbol`, `trade_count`, `win_count`, `win_rate`, `total_pnl`, `avg_holding_days`, `first_trade_date`, `last_trade_date`
- 按总盈亏降序排列

- Schema: 在 `StatsResponse` 新增 `symbol_summary: list[SymbolSummaryItem]`
- `SymbolSummaryItem`: 上述字段

**Frontend** — 新建 `frontend/src/components/SymbolSummaryTable.tsx`

- 可排序表格：股票代码、交易次数、胜率、总盈亏（红绿色）、平均持仓
- 放在 StatsTab 核心结果下方、进阶分析上方
- 默认按总盈亏降序

### Task 3: 扩充 AI Prompt

**Backend** — `backend/app/api/report.py` + `backend/app/ai/prompt.py` + `backend/app/ai/validator.py`

修改 `_build_analysis_data()` 在 `report.py`：
- 接收 `stats` 数据（从 analysis.py 的 get_stats 逻辑复用或直接计算）
- 新增字段: `profit_factor`, `expectancy`, `max_drawdown`, `max_drawdown_pct`, `consecutive_losses`, `avg_mae`, `avg_mfe`, `profit_capture_ratio`, `win_loss_ratio`, `total_return_pct`
- 新增 `positions_summary`: 盈利 TOP3 + 亏损 TOP3（symbol, pnl, pnl_pct, holding_days, patterns）

修改 `build_user_prompt()` 在 `prompt.py`：
- 将新字段格式化到 prompt 文本中
- 结构化呈现：风险指标块 + 行为指标块 + 关键交易块

修改 `validator.py`：
- 新增对 `profit_factor`, `max_drawdown_pct`, `consecutive_losses` 的数值校验（±1% 容忍度）

**Tests** — 更新 `backend/tests/test_ai/test_prompt.py` 和 `test_validator.py`

## 5. Technical Notes

- 净值曲线数据在 get_stats() 中已隐式计算（max_dd 循环），只需收集数据点
- 股票维度分组使用 `valid_positions`（cost_known=True），与 KPI 口径一致
- AI Prompt 扩充需要 `_build_analysis_data()` 能访问到 stats 级别的指标，当前 report.py 的 generate_report() 已有 positions/market_data/insight/whatif，需要补充 stats 级计算
- Recharts 已在 package.json 中，无需新增依赖
- 所有显示文本中文优先（CLAUDE.md UI 规范）
- 小样本（<5 笔）不评价，显示"样本不足"（已有 is_small_sample 字段）
