## 问题 1：分析面板是否达到行业主流水平？

### 评分：8 / 10

### 1.1 指标覆盖与计算正确性

计算口径完全正确且行业对齐的指标：

| 指标 | 计算方法 | 与 TradesViz 对齐 | 评价 |
|------|---------|-----------------|------|
| 胜率 | pnl>0 计数/总笔数，pnl==0 计入亏损 | ✅ 对齐 | 精确到含费后确为微亏 |
| 盈亏比(PF) | gross_profit / gross_loss | ✅ 对齐 | 正确拒绝 win_rate/(1-win_rate) 的常见错误 |
| 预期收益(Expectancy) | R-multiple，基于 pnl_pct | ✅ 对齐 | 正确拒绝基于绝对金额的版本 |
| 最大回撤 | % + 绝对金额 | ✅ 对齐 | 峰值从 0 开始，连续亏损场景有兜底 |
| 损益比(Payoff Ratio) | avg_win / avg_loss | ✅ 对齐 | 无亏损时返回 None 而非 0 |
| MAE/MFE | 日线 low/high | ✅ 对齐 | 行业标准 |
| 止盈效率(Profit Capture) | mean(per-position pnl/MFE) | ✅ 对齐 | 正确使用均值而非 sum/sum |

### 1.2 止损回测的正确性（加分项）

`whatif.py` 中存在三处关键的专业设计，只有 TradesViz Pro 级别工具才做：

1. **日内触发检查**：遍历持仓期间的日线 low，判断是否盘中触及止损线，不是简单截断最终 PnL
2. **跳空低开处理**：`fill_price = min(open, stop_price)`，正确模拟开盘跳空低开止损按开盘价成交
3. **手续费保持**：触发止损的仓位仍扣除真实佣金，防止"止损后收益看起来更好"的假象

### 1.3 独特能力——行业领先

**Shapley Value 公平归因**是该项目相对于 TradesViz/Edgewonk/Tradervue 的独特优势：

- TradesViz 的归因逻辑：按标签分组 → 汇总组内 PnL。当一笔交易有多个标签时，各组 PnL 之和 ≠ 总 PnL
- 本项目用 Shapley Value（`attribution.py`）通过蒙特卡洛采样公平分配联合贡献，保证各标签贡献之和 = 总 PnL，消除了归因重叠问题
- 这是学术界级别的归因方案，当前没有任何主流零售复盘工具提供

**AI 诊断报告 + 校验器**也是差异化能力。TradesViz 有自动标注但没有 AI 生成的自然语言报告。`validator.py` 用正则提取 AI 输出中的关键数字与输入数据比对（±1% 容忍度），防止 AI 幻觉。

### 1.4 与主流工具的差距（扣分项）

| 缺失特性 | TradesViz | Edgewonk | 严重程度 | 说明 |
|---------|-----------|----------|---------|------|
| 净值曲线图 | 有 | 有 | 🔴 高 | 最基础的视觉工具 |
| 股票维度盈亏 | 有 | 有 | 🔴 高 | 散户最关心"哪个股票赚/亏了钱" |
| 日历热力图 | 有 | 无 | 🟡 中 | TradesViz 的招牌功能 |
| 基准对比 | 有 | 有 | 🟡 中 | 无法对比沪深300同期表现 |
| 滚动窗口统计 | 有 | 有 | 🟡 中 | 无法看"近期 30 天表现趋势" |
| 交易笔记功能 | 有 | 有 | 🟡 中 | 无法标记某笔交易的特殊情况 |
| 夏普/索提诺比率 | 有 | 有 | 🟢 低 | 对散户意义有限 |
| 日内时段分析 | 有 | 无 | 🟢 低 | TradesViz 有"周几表现好"分析 |

**最关键缺失**：没有**股票维度**的盈亏拆解。散户最基本的问题是"我在哪只股票上赚了亏了？"——当前产品只能通过 Positions 表格逐行查看，没有汇总。

## 问题 2：指标是否过于专业，散户看不懂？

### 总体判断：信息分层做得很好，但仍有 3 个指标存在显著认知障碍

项目方对散户认知问题的重视程度超出行业平均水平。三种关键设计决策正确：

1. **三层信息架构**：核心结果 → 进阶分析 → 高级分析（折叠）
2. **每个指标有评级**：散户不需要自己判断"2.3 的 PF 好不好"，系统直接标出
3. **Shapley 默认折叠**：最复杂的归因被折叠，"赚钱来源分析（公平归因，点击展开）"

### 2.1 认知分层评估

**第一层：散户直觉能理解（零障碍）**
- 总盈亏（金额+收益率%）、胜率、单笔最大盈利/亏损、完整交易笔数、平均持仓天数、连续亏损次数

**第二层：需要一句话解释（轻度障碍）**
- 盈亏比(PF)：附评级，散户能理解
- 最大回撤：有级别标注，但需要一句"从最高点跌下来多少"的解释
- 损益比(Payoff Ratio)：当前无解释文字，散户会困惑"这个和 PF 有什么区别？"

**第三层：存在显著认知障碍**

| 问题指标 | 散户困惑 | 建议修复 |
|----------|----------|----------|
| **Expectancy（预期收益）** | 误解为"平均每笔收益率"，实际是统计期望值。散户会想"每笔预期赚 1%，100 笔赚 100%？但实际不是啊。" | 加脚注「※ 预期收益≠平均每笔收益率。它是胜率×平均盈利比率 − 败率×平均亏损比率的加权结果，衡量交易系统的数学期望。」 |
| **MAE（最大回撤容忍度）** | 与 Max Drawdown 混淆，分不清"账户回撤"和"单笔持仓浮亏" | 改名「持仓浮亏容忍度」，解释改为"你在持仓过程中，账面上最多亏到过多少（不是最终的盈亏）" |
| **止盈效率（Profit Capture）** | "40% 是好是差？我该做什么？" | 改名「浮盈兑现率」，解释改为"你买到的股票涨了 100%，你只赚到 40% 就卖了——这是你'拿不住'的程度。" |
| **PF vs Payoff Ratio** | 两个都是"盈亏比" | UI 上加差异化：PF="总盈利÷总亏损" vs Payoff="平均每笔盈利÷平均亏损" |

### 2.2 改进建议汇总

| 问题 | 优先级 | 建议 |
|------|--------|------|
| PF 和 Payoff Ratio 混淆 | 🔴 高 | UI 加差异化解释 |
| Expectancy 被误解 | 🔴 高 | 加脚注或模拟案例说明 |
| MAE 与 Max Drawdown 混淆 | 🟡 中 | 改名"持仓浮亏容忍度" |
| 缺少"收益 vs 风险"心智模型 | 🟡 中 | 参考 TradesViz 加收益-风险仪表盘 |
| 缺少 so-what 文本 | 🟡 中 | 每个指标加"这意味着什么"（如止盈效率 0.2 → "你只兑现了 20% 的浮盈，说明经常过早卖出"） |

## 问题 3：是否达到了「让用户知道自己为什么亏钱、为什么赚钱」的设计目标？

### 总体判断：部分达成但闭环不完整

### 3.1 正确的部分

**行为标签系统质量较高**（`pattern.py`）：

- `CHASE`（追涨）：检查 5 日涨幅 >15% + 入场价 > MA20×1.10 + 入场价 ≥ 20日高点×0.97
- `BOTTOM`（抄底）：检查 5 日跌幅 >15% + MA20 < MA60（下跌趋势）
- `PYRAMID`（加仓）：已有持仓时以更高价格买入
- `AVERAGE_DOWN`（补仓）：前序持仓亏损 + 补仓价低于首次买入均价×0.95
- 4 维度清晰区分：市场环境 ≠ 交易行为 ≠ 交易结果 ≠ 心理推测

**Shapley 归因消除了重复计算**：蒙特卡洛采样在模式集合的边际贡献层面分配 PnL，sum = total_pnl，解决了 TradesViz 的分组求和导致各组之和不等于总 PnL 的问题。

**止损回测的结论文本**：从数据自动生成「历史数据支持 5% 止损」/「5% 止损会严重损害收益，建议 8-10%」/「影响极小，作为辅助风控手段即可」——直接告诉散户止损规则是否适合自己。

### 3.2 缺失的部分

| 能力 | 达成度 | 说明 |
|------|--------|------|
| 用户能看到自己的行为模式 | 70% | 4 维度标签+统计覆盖主要行为，但缺少跨维度组合和股票维度的分析 |
| 归因分析回答"为什么赚钱/亏钱" | 75% | Shapley 和因子移除方案正确，多维度交叉分析不足 |
| AI 诊断连接数据与行动建议 | 50% | Prompt 信息量不足限制了 AI 的深度 |
| 时间线叙事 | 10% | 心理标签检测了但没有时间线可视化 |
| 从"知道"到"行动" | 0% | 完全缺失 |

**缺失 1：AI Prompt 信息量不足（关键）**

当前 `build_user_prompt()` 只发送了以下字段到 AI：

```
total_trades, win_rate, total_pnl, avg_holding_days,
patterns[].{name, count, win_rate, total_pnl},
what_if[].{pattern, delta, contribution_pct}
```

**未发送到 AI 的数据**：MAE、MFE、止盈效率、PF、Expectancy、Payoff Ratio（每模式级别）、最大回撤、连续亏损、交易明细（positions）

这意味着 AI 生成的诊断报告不知道用户的最大回撤是 30% 还是 3%，无法针对具体交易给出建议。

**缺失 2：没有跨维度交叉分析**

当前 Insight 面板每个维度独立统计，但散户最关心的组合模式看不到：
- "在熊市追涨"的表现如何？
- "追涨 + 短线"同时出现的交易表现如何？

**缺失 3：没有时间线叙事**

心理标签（POSSIBLE_REVENGE、OVERTRADING、HOLD_LOSER、CUT_WINNER）已检测但在前端仅以表格展示，无法串联成"你在大亏后开始报复性交易"这样的叙事链。

**缺失 4：没有"行动计划"闭环**

AI 报告的最后是"改善建议"，但没有功能让用户把这些建议转化为行动：
- "建议减少追涨" → 没有"设定行为目标"的功能
- "建议设置 5% 止损" → 没有"启用止损提醒"的机制

设计目标止步于"分析"阶段，没有进入"行为改善"阶段。

## 优先级路线图

### 短期（1-2 周）

1. 🔴 **净值曲线图**：最基础的视觉工具
2. 🔴 **股票维度盈亏表**：散户最基本的需求——"哪只股票赚了亏了"
3. 🔴 **扩充 AI Prompt**：将 MAE/MFE/PF/Expectancy/最大回撤/连续亏损送入 AI

### 中期（1-2 月）

4. 🟡 **时间线叙事**：心理标签 + What-If + 时间线 = 完整行为叙事
5. 🟡 **指标"人话"脚注**：Expectancy、MAE、止盈效率的散户友好解释
6. 🟡 **行动计划机制**：从分析到行动，设定行为改善目标
7. 🟡 **跨维度交叉分析**：组合模式统计（如"熊市+追涨"）

### 长期（3-6 月）

8. 🟢 **日历热力图**：TradesViz 招牌功能
9. 🟢 **滚动窗口统计**：近期 N 天表现趋势
10. 🟢 **基准对比**：与沪深 300 同期对比
11. 🟢 **交易日记功能**：标记特殊情况

## 总评

TradingJournalAnalyzer 当前是一个**接近行业主流水平的 A 股散户交易复盘工具**。核心计算引擎（Expectancy、PF、MAE/MFE、止损回测）严格对齐 TradesViz/Edgewonk/Tradervue 标准，Shapley Value 归因和 AI 诊断报告是两个超出行业水平的差异化能力。

核心短板在于：
1. **视觉化呈现不够**：缺少净值曲线和股票维度分析
2. **AI 报告信息量不足**：Prompt 只送了 5 个字段
3. **闭环不完整**：用户知道问题但不知道下一步该做什么

补齐这些短板后，产品可以达到甚至部分超越 TradesViz 对 A 股散户的适配度。
